## The common pattern

Every finding across all three campaigns is the **same root cause**: when a session ends or pauses, the *fast-moving* files (the chronicle, the NPC files actively in a scene) get updated, but the *slow* orientation files — chiefly **`meta/calendar.md`** and a few **`world/`** files — get left behind. They're the files a fresh chat reads *first* (§2 boot set, ★), so stale ones do the most damage: they orient a new session to the wrong place.

**The single highest-value habit to adopt:** on every `/save` (and ideally on `/endscene`), re-touch `meta/calendar.md` and any `world/` file whose facts the just-played scene changed. The chronicle and NPC files are already getting this attention; the calendar and world files are the gap.

A secondary pattern worth borrowing from **the-thaw**: it keeps a **canon-correction log** (in `tone-and-rules.md`) and **self-flags TBDs** ("name not yet established… to be canonised on next mention") rather than letting the prose invent a fact. bonefern and phoenix-claw would both benefit from the same discipline.
